1. NUCLEO-G431KB의 CAN 컨트롤러 여부

FDCAN 컨트롤러가 내장되어 있습니다. STM32G431KB는 FDCAN1을 내장하고 있어서 별도의 MCP2515 같은 외부 CAN 컨트롤러가 불필요합니다.

따라서 구성은:

STM32G431KB (FDCAN 내장) → SN65HVD230 (트랜시버만) → CAN 버스

이것으로 충분합니다. MCP2551은 사용할 필요 없습니다.

    ⚠️ MCP2551은 5V용 트랜시버인데, STM32G431KB는 3.3V 로직이라 SN65HVD230(3.3V)이 올바른 선택입니다.

2. 회로도 검토

회로도를 보면 전체적인 구성은 합리적이나 몇 가지 확인/수정이 필요한 부분이 있습니다.
✅ 올바른 부분

    LCD I2C 연결 (SDA/SCL)
    MG966R 서보 3개 + SG90 서보 3개 구성
    5V/3.3V 전원 분리

⚠️ 확인/수정 필요한 부분

① CAN 종단 저항 누락

CANH ---[120Ω]--- CANL  (버스 양 끝단에 각각 필요)

회로도에 종단 저항(120Ω)이 보이지 않습니다. CAN 버스 양쪽 끝에 반드시 추가하세요.

② SN65HVD230의 RS 핀 처리

    RS 핀(핀 8)은 GND에 연결해야 고속 모드로 동작합니다.
    회로도에서 확인이 어렵습니다.

③ 서보모터 전원

    MG966R은 최대 2.5A씩 소모 → 3개면 최대 7.5A
    반드시 별도 외부 전원(5V/10A급)에서 공급해야 합니다.
    STM32 보드의 5V 핀으로 서보에 전원을 공급하면 안 됩니다.
    GND는 반드시 MCU GND와 공통 연결

④ NUCLEO-G431KB의 실제 사용 가능 핀 확인

    NUCLEO-G431KB는 LQFP32 패키지로 핀 수가 적습니다
    PWM 6채널 + I2C + FDCAN + UART + ADC(밴딩센서) 를 동시에 쓰면 핀이 빠듯합니다

2. USE_COUNTING_SEMAPHORES: Disabled → Enabled
Semaphore(세마포어)가 뭔가요?
쉽게 말하면 "지금 데이터 있어요!" 신호를 주고받는 깃발입니다.
[CAN 수신 인터럽트]
    CAN 데이터 도착!
         ↓
    세마포어 Give (깃발 올림)
         ↓
[CAN_Task]
    세마포어 Take (깃발 확인)
         ↓
    데이터 처리!
Binary vs Counting 차이
종류동작사용 상황Binary Semaphore0 또는 1 (있다/없다)단순 신호Counting Semaphore0,1,2,3... (몇 개?)여러 이벤트 누적
왜 Counting이 필요한가요?
CAN 메시지가 빠르게 3개 연속 도착했을 때:

Binary:   신호 1개만 기록 → 2개 유실 ❌
Counting: 신호 3개 기록  → 3개 모두 처리 ✅
CAN 통신처럼 연속으로 데이터가 올 수 있는 상황에서 필요합니다.

1. 클럭 소스 (Clock Source)
MCU에게 박자를 제공하는 원천입니다.
HSI (High Speed Internal)
MCU 칩 내부에 내장된 클럭
├── 장점: 외부 부품 불필요, 바로 사용 가능
├── 단점: 정확도 낮음 (온도따라 약간 오차)
└── 현재 설정: 16MHz
HSE (High Speed External)
외부에 달린 크리스탈 oscillator (수정진동자)
├── 장점: 정확도 매우 높음
├── 단점: 외부 부품 필요
└── NUCLEO-G431KB: 24MHz 크리스탈 내장
    → 현재 이게 선택되어 있음 ✅
LSI (Low Speed Internal)
내부 저속 클럭
├── 용도: RTC, 워치독 타이머 전용
└── 속도: 32KHz (매우 느림)
LSE (Low Speed External)
외부 저속 크리스탈
├── 용도: RTC (정확한 시간 측정)
└── 속도: 32.768KHz

3. Prescaler, Time Seg1, Time Seg2, SJW
이 4개는 모두 Baudrate(통신 속도)를 결정하는 값들입니다.
CAN Bit 구조 이해
CAN은 1bit를 여러 구간으로 나눠서 전송합니다

1 bit = [Sync] + [Time Seg1] + [Time Seg2]
         1TQ      9TQ           10TQ
         
TQ(Time Quantum) = 클럭의 최소 단위
Prescaler
FDCAN 클럭(170MHz)을 나눠서 TQ 크기 결정

Prescaler = 17
TQ = 170MHz / 17 = 10MHz
   = 0.1μs (TQ 하나의 길이)
Time Seg1
샘플링 포인트 이전 구간
= 신호가 안정화되길 기다리는 시간

Time Seg1 = 9TQ
= 9 × 0.1μs = 0.9μs
Time Seg2
샘플링 포인트 이후 구간
= 다음 bit 준비 시간

Time Seg2 = 10TQ
= 10 × 0.1μs = 1.0μs
최종 Baudrate 계산
1 bit = Sync(1) + Seg1(9) + Seg2(10) = 20 TQ

Baudrate = TQ 클럭 / 1bit당 TQ 수
         = 10MHz / 20
         = 500kbps ✅

4. SJW (Synchronization Jump Width)
송신측과 수신측의 클럭이 완벽히 같을 수 없음
→ 약간의 오차 발생

SJW = 이 오차를 보정할 수 있는 최대 TQ 수

SJW = 1
→ 최대 1TQ 만큼 오차 보정 가능
= 0.1μs 오차까지 허용

일반적으로 1~4 사이 값 사용
500kbps에서는 1로 충분 ✅

전체 그림으로 이해
1 bit 전송 구조 (20 TQ)

|sync|----Seg1(9TQ)----|----Seg2(10TQ)----|
  ↑                  ↑
시작              여기서 신호값 읽음
                  (샘플링 포인트)

Prescaler 17로 나눠서
TQ = 0.1μs

20 TQ × 0.1μs = 2μs = 1bit
1bit = 2μs → 500,000 bps = 500kbps ✅

MCU = 회사

RCC = 총무부서
       ├── 전기 공급 관리 (클럭 공급)

RCC의 주요 역할
1. 클럭 소스 관리
어떤 클럭을 사용할지 선택

각 주변장치에 클럭을 줄지 말지 결정

RCC → UART 클럭 공급 OFF → UART 동작 안함
                            (전력 절약)
3. 리셋 관리
├── 시스템 전체 리셋
├── 각 주변장치 개별 리셋
└── 리셋 원인 기록

/****************************************************************
    Function: CAN_Transmit
    Description: 리모콘 → 로봇팔 데이터 송신

    [0~1] bendingSensor[0]
    [2~3] bendingSensor[1]
    [4~5] gyro_pitch
    [6~7] gyro_roll
    * checksum은 다음 프레임에 포함
****************************************************************/

    FDCAN_TxHeaderTypeDef txHeader;
    uint8_t txBuffer[8] = {0};

    // 데이터 패킹
    txBuffer[0] = (txData->bendingSensor[0] >> 8) & 0xFF;
    txBuffer[1] =  txData->bendingSensor[0] & 0xFF;
    txBuffer[2] = (txData->bendingSensor[1] >> 8) & 0xFF;
    txBuffer[3] =  txData->bendingSensor[1] & 0xFF;
    txBuffer[4] = (txData->gyro_pitch >> 8) & 0xFF;
    txBuffer[5] =  txData->gyro_pitch & 0xFF;
    txBuffer[6] = (txData->gyro_roll >> 8) & 0xFF;
    txBuffer[7] =  txData->gyro_roll & 0xFF;

    // CAN 헤더 설정
    txHeader.Identifier          = CAN_ID_REMOTE_DATA;
    txHeader.IdType              = FDCAN_STANDARD_ID;
    txHeader.TxFrameType         = FDCAN_DATA_FRAME;
    txHeader.DataLength          = FDCAN_DLC_BYTES_8;
    txHeader.ErrorStateIndicator = FDCAN_ESI_ACTIVE;

    txHeader.TxEventFifoControl  = FDCAN_NO_TX_EVENTS;

uint8_t CAN_IsConnected(void) {
    if((HAL_GetTick() - lastCanRxTick) < CAN_TIMEOUT_MS) {
        return 1;
    }
    canConnected = 0;
    return 0;
}

uint8_t CAN_CalcChecksum(uint8_t *data, uint8_t length) {
    uint8_t sum = 0;
    for(int i = 0; i < length; i++) {
        sum += data[i];
    }
    return sum;
}

// 리모콘 데이터 송신
// RemoteTxData는 8byte 초과이므로 2개 프레임으로 분할
HAL_StatusTypeDef CAN_TransmitRemoteData(RemoteTxData *txData) {
    FDCAN_TxHeaderTypeDef txHeader;
    uint8_t txBuffer[8] = {0};
    HAL_StatusTypeDef ret;

    txHeader.Identifier          = CAN_ID_REMOTE;
    txHeader.IdType              = FDCAN_STANDARD_ID;
    txHeader.TxFrameType         = FDCAN_DATA_FRAME;
    txHeader.DataLength          = FDCAN_DLC_BYTES_8;
    txHeader.ErrorStateIndicator = FDCAN_ESI_ACTIVE;
    txHeader.BitRateSwitch       = FDCAN_BRS_OFF;
    txHeader.FDFormat            = FDCAN_CLASSIC_CAN;
    txHeader.TxEventFifoControl  = FDCAN_NO_TX_EVENTS;
    txHeader.MessageMarker       = 0;

    // 프레임 1: 센서 데이터
    // [0~1] bendingSensor[0]
    // [2~3] bendingSensor[1]
    // [4~5] gyro_pitch
    // [6~7] gyro_roll
    txBuffer[0] = (txData->bendingSensor[0] >> 8) & 0xFF;
    txBuffer[1] =  txData->bendingSensor[0] & 0xFF;
    txBuffer[2] = (txData->bendingSensor[1] >> 8) & 0xFF;
    txBuffer[3] =  txData->bendingSensor[1] & 0xFF;
    txBuffer[4] = (txData->gyro_pitch >> 8) & 0xFF;
    txBuffer[5] =  txData->gyro_pitch & 0xFF;
    txBuffer[6] = (txData->gyro_roll >> 8) & 0xFF;
    txBuffer[7] =  txData->gyro_roll & 0xFF;

    txData->checksum = CAN_CalcChecksum(txBuffer, 8);
    ret = HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
              &txHeader, txBuffer);
    if(ret != HAL_OK) return ret;

    // 프레임 2: 시스템 상태
    // [0] remoteCommStatus
    // [1] remoteSysVolt
    // [2] remoteSensorStatus
    // [3] checksum
    txHeader.Identifier = CAN_ID_REMOTE + 1;  // 0x101
    txBuffer[0] = txData->remoteCommStatus;
    txBuffer[1] = txData->remoteSysVolt;
    txBuffer[2] = txData->remoteSensorStatus;
    txBuffer[3] = txData->checksum;
    txBuffer[4] = 0;
    txBuffer[5] = 0;
    txBuffer[6] = 0;
    txBuffer[7] = 0;

    return HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
               &txHeader, txBuffer);
}

HAL_StatusTypeDef CAN_TransmitRobotData(RobotTxData *txData) {
    FDCAN_TxHeaderTypeDef txHeader;
    uint8_t txBuffer[8] = {0};
    HAL_StatusTypeDef ret;

    txHeader.Identifier          = CAN_ID_ROBOT;
    txHeader.IdType              = FDCAN_STANDARD_ID;
    txHeader.TxFrameType         = FDCAN_DATA_FRAME;
    txHeader.DataLength          = FDCAN_DLC_BYTES_8;
    txHeader.ErrorStateIndicator = FDCAN_ESI_ACTIVE;
    txHeader.BitRateSwitch       = FDCAN_BRS_OFF;
    txHeader.FDFormat            = FDCAN_CLASSIC_CAN;
    txHeader.TxEventFifoControl  = FDCAN_NO_TX_EVENTS;
    txHeader.MessageMarker       = 0;

    // 프레임 1: 모터 각도 1~4
    txBuffer[0] = (txData->motorAngle[0] >> 8) & 0xFF;
    txBuffer[1] =  txData->motorAngle[0] & 0xFF;
    txBuffer[2] = (txData->motorAngle[1] >> 8) & 0xFF;
    txBuffer[3] =  txData->motorAngle[1] & 0xFF;
    txBuffer[4] = (txData->motorAngle[2] >> 8) & 0xFF;
    txBuffer[5] =  txData->motorAngle[2] & 0xFF;
    txBuffer[6] = (txData->motorAngle[3] >> 8) & 0xFF;
    txBuffer[7] =  txData->motorAngle[3] & 0xFF;

    ret = HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
              &txHeader, txBuffer);
    if(ret != HAL_OK) return ret;

    // 프레임 2: 모터 각도 5~6 + 시스템 상태
    txHeader.Identifier = CAN_ID_ROBOT + 1;  // 0x201
    txBuffer[0] = (txData->motorAngle[4] >> 8) & 0xFF;
    txBuffer[1] =  txData->motorAngle[4] & 0xFF;
    txBuffer[2] = (txData->motorAngle[5] >> 8) & 0xFF;
    txBuffer[3] =  txData->motorAngle[5] & 0xFF;
    txBuffer[4] =  txData->robotCommStatus;
    txBuffer[5] =  txData->robotSysVolt;
    txBuffer[6] =  txData->motorStatus;
    txBuffer[7] =  txData->lcdStatus;

    return HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
               &txHeader, txBuffer);
}

uint8_t CAN_ParseRemoteData(CAN_RxRaw_t *rxRaw,
                             RemoteTxData *payload) {
    if(rxRaw->id == CAN_ID_REMOTE) {
        // 프레임 1 파싱
        payload->bendingSensor[0] =
            (rxRaw->data[0] << 8) | rxRaw->data[1];
        payload->bendingSensor[1] =
            (rxRaw->data[2] << 8) | rxRaw->data[3];
        payload->gyro_pitch =
            (rxRaw->data[4] << 8) | rxRaw->data[5];
        payload->gyro_roll  =
            (rxRaw->data[6] << 8) | rxRaw->data[7];
        return 1;
    }
    else if(rxRaw->id == CAN_ID_REMOTE + 1) {
        // 프레임 2 파싱
        payload->remoteCommStatus  = rxRaw->data[0];
        payload->remoteSysVolt     = rxRaw->data[1];
        payload->remoteSensorStatus= rxRaw->data[2];
        payload->checksum          = rxRaw->data[3];
        return 1;
    }
    return 0;
}

uint8_t CAN_ParseRobotData(CAN_RxRaw_t *rxRaw,
                            RobotTxData *payload) {
    if(rxRaw->id == CAN_ID_ROBOT) {
        // 프레임 1 파싱
        payload->motorAngle[0] =
            (rxRaw->data[0] << 8) | rxRaw->data[1];
        payload->motorAngle[1] =
            (rxRaw->data[2] << 8) | rxRaw->data[3];
        payload->motorAngle[2] =
            (rxRaw->data[4] << 8) | rxRaw->data[5];
        payload->motorAngle[3] =
            (rxRaw->data[6] << 8) | rxRaw->data[7];
        return 1;
    }
    else if(rxRaw->id == CAN_ID_ROBOT + 1) {
        // 프레임 2 파싱
        payload->motorAngle[4] =
            (rxRaw->data[0] << 8) | rxRaw->data[1];
        payload->motorAngle[5] =
            (rxRaw->data[2] << 8) | rxRaw->data[3];
        payload->robotCommStatus = rxRaw->data[4];
        payload->robotSysVolt    = rxRaw->data[5];
        payload->motorStatus     = rxRaw->data[6];
        payload->lcdStatus       = rxRaw->data[7];
        return 1;
    }
    return 0;
}

            txHeader.Identifier = CAN_ID_ROBOT;

            txBuffer[1] =  txData->motorAngle[0] & 0xFF;
            txBuffer[2] = (txData->motorAngle[1] >> 8) & 0xFF;
            txBuffer[3] =  txData->motorAngle[1] & 0xFF;
            txBuffer[4] = (txData->motorAngle[2] >> 8) & 0xFF;
            txBuffer[5] =  txData->motorAngle[2] & 0xFF;
            txBuffer[6] = (txData->motorAngle[3] >> 8) & 0xFF;
            txBuffer[7] =  txData->motorAngle[3] & 0xFF;

            ret = HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
                      &txHeader, txBuffer);
            if(ret != HAL_OK) return ret;

            // 프레임 2: 모터 5~6 + 시스템 상태 (0x201)
            txHeader.Identifier = CAN_ID_ROBOT + 1;
            txBuffer[0] = (txData->motorAngle[4] >> 8) & 0xFF;
            txBuffer[1] =  txData->motorAngle[4] & 0xFF;
            txBuffer[2] = (txData->motorAngle[5] >> 8) & 0xFF;
            txBuffer[3] =  txData->motorAngle[5] & 0xFF;
            txBuffer[4] =  txData->robotCommStatus;
            txBuffer[5] =  txData->robotSysVolt;
            txBuffer[6] =  txData->motorStatus;
            txBuffer[7] =  txData->lcdStatus;

            return HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
                       &txHeader, txBuffer);

        default:
            return HAL_ERROR;
    }
}

/****************************************************************
    Function: CAN_Parse
    Description: 수신 ID로 switch-case 파싱
****************************************************************/
uint8_t CAN_Parse(CAN_RxRaw_t *rxRaw, TxData *payload) {
    switch(rxRaw->id) {
        // 리모콘 데이터 파싱
        case CAN_ID_REMOTE:
            payload->sourceId         = CAN_ID_REMOTE;
            payload->bendingSensor[0] =
                (rxRaw->data[0] << 8) | rxRaw->data[1];
            payload->bendingSensor[1] =
                (rxRaw->data[2] << 8) | rxRaw->data[3];
            payload->gyro_pitch =
                (rxRaw->data[4] << 8) | rxRaw->data[5];
            payload->gyro_roll  =
                (rxRaw->data[6] << 8) | rxRaw->data[7];
            return 1;

        case CAN_ID_REMOTE + 1:  // 0x101
            payload->remoteCommStatus   = rxRaw->data[0];
            payload->remoteSysVolt      = rxRaw->data[1];
            payload->remoteSensorStatus = rxRaw->data[2];
            payload->checksum           = rxRaw->data[3];
            return 1;

        // 로봇팔 데이터 파싱
        case CAN_ID_ROBOT:
            payload->sourceId      = CAN_ID_ROBOT;
            payload->motorAngle[0] =
                (rxRaw->data[0] << 8) | rxRaw->data[1];
            payload->motorAngle[1] =
                (rxRaw->data[2] << 8) | rxRaw->data[3];
            payload->motorAngle[2] =
                (rxRaw->data[4] << 8) | rxRaw->data[5];
            payload->motorAngle[3] =
                (rxRaw->data[6] << 8) | rxRaw->data[7];
            return 1;

        case CAN_ID_ROBOT + 1:  // 0x201
            payload->motorAngle[4] =
                (rxRaw->data[0] << 8) | rxRaw->data[1];
            payload->motorAngle[5] =
                (rxRaw->data[2] << 8) | rxRaw->data[3];
            payload->robotCommStatus = rxRaw->data[4];
            payload->robotSysVolt    = rxRaw->data[5];
            payload->motorStatus     = rxRaw->data[6];
            payload->lcdStatus       = rxRaw->data[7];
            return 1;

        default:
            return 0;  // 알 수 없는 ID
    }
}

    uint8_t sum = 0;

    Description: CAN 수신 인터럽트 콜백
                 ID 범위로 분기
****************************************************************/
void HAL_FDCAN_RxFifo0Callback(FDCAN_HandleTypeDef *hfdcan,
                                uint32_t RxFifo0ITs) {
    if(hfdcan->Instance == FDCAN1) {
        FDCAN_RxHeaderTypeDef rxHeader;
        uint8_t rxData[8] = {0};

        HAL_FDCAN_GetRxMessage(hfdcan,
            FDCAN_RX_FIFO0, &rxHeader, rxData);

        lastCanRxTick = HAL_GetTick();
        canConnected  = 1;

        // ID 범위로 분기
        if(rxHeader.Identifier >= 0x100 &&
           rxHeader.Identifier <= 0x1FF) {
            // 리모콘 데이터
            Fill_Remote_CAN_Message(rxHeader.Identifier, rxData);

        } else if(rxHeader.Identifier >= 0x200 &&
                  rxHeader.Identifier <= 0x2FF) {
            // 로봇팔 데이터
            Fill_Robot_CAN_Message(rxHeader.Identifier, rxData);
        }
    }
}

1. Checksum이란?
데이터 전송 중 오류가 생겼는지 검출하는 값

송신측:
데이터 [10, 20, 30, 40] 합산
→ checksum = 10+20+30+40 = 100
→ 데이터와 함께 전송

수신측:
받은 데이터 [10, 20, 30, 40] 합산
→ 계산값 = 100
→ 받은 checksum = 100
→ 일치! → 정상 수신 ✅

노이즈로 데이터 손상:
받은 데이터 [10, 25, 30, 40] 합산
→ 계산값 = 105
→ 받은 checksum = 100
→ 불일치! → 오류 감지 ❌

2. 인터럽트 콜백 함수는 필요한가?
필요합니다!

CAN 메시지가 언제 올지 모름
→ 콜백이 없으면 계속 폴링해야 함
→ CPU 낭비

콜백 방식:
메시지 도착 → 하드웨어가 자동으로
              HAL_FDCAN_RxFifo0Callback() 호출
→ Fill 함수 호출
→ 구조체에 저장
→ Task는 구조체만 읽으면 됨

/****************************************************************
    Function: CAN_Start
****************************************************************/
void CAN_Start(void) {
    FDCAN_FilterTypeDef filterConfig;

    // 리모콘 데이터 범위 (0x100~0x1FF)
    filterConfig.IdType       = FDCAN_STANDARD_ID;
    filterConfig.FilterIndex  = 0;
    filterConfig.FilterType   = FDCAN_FILTER_RANGE;
    filterConfig.FilterConfig = FDCAN_FILTER_TO_RXFIFO0;
    filterConfig.FilterID1    = 0x100;
    filterConfig.FilterID2    = 0x1FF;
    HAL_FDCAN_ConfigFilter(&hfdcan1, &filterConfig);

    // 로봇팔 데이터 범위 (0x200~0x2FF)
    filterConfig.FilterIndex  = 1;
    filterConfig.FilterID1    = 0x200;
    filterConfig.FilterID2    = 0x2FF;
    HAL_FDCAN_ConfigFilter(&hfdcan1, &filterConfig);

    HAL_FDCAN_ActivateNotification(&hfdcan1,
        FDCAN_IT_RX_FIFO0_NEW_MESSAGE, 0);
    HAL_FDCAN_Start(&hfdcan1);
}

/****************************************************************
    Function: CAN_IsConnected
****************************************************************/
uint8_t CAN_IsConnected(void) {
    if((HAL_GetTick() - lastCanRxTick) < CAN_TIMEOUT_MS) {
        return 1;
    }
    canConnected = 0;
    return 0;
}

/****************************************************************
    Function: CAN_CalcChecksum
****************************************************************/
uint8_t CAN_CalcChecksum(uint8_t *data, uint8_t length) {
    uint8_t sum = 0;
    uint8_t i;
    for(i = 0; i < length; i++) {
        sum += data[i];
    }
    return sum;
}

/****************************************************************
    Function: Fill_Remote_CAN_Message
    Description: 리모콘 RAW CAN 데이터 → RemoteCanMsg 채우기
****************************************************************/
void Fill_Remote_CAN_Message(uint32_t canID, uint8_t rxData[8]) {
    uint8_t i;

    switch(canID) {
        case CAN_ID_REMOTE_SENSOR:   // 0x100
            for(i = 0; i < 8; i++) {
                RemoteCanMsg.REMOTE_SENSOR.BYTE_FIELD[i] = rxData[i];
            }
            DIAG_MsgRxCnt_Remote++;
            break;

        case CAN_ID_REMOTE_STATUS:   // 0x102
            for(i = 0; i < 8; i++) {
                RemoteCanMsg.REMOTE_STATUS.BYTE_FIELD[i] = rxData[i];
            }
            DIAG_MsgRxCnt_Remote++;
            break;

        default:
            break;
    }

    // RAW 채운 후 공용 구조체 업데이트
    Update_RemoteData();
}

/****************************************************************
    Function: Tx_Remote_CAN_Message
    Description: 리모콘 데이터 송신
****************************************************************/
HAL_StatusTypeDef Tx_Remote_CAN_Message(uint32_t canID) {
    uint8_t i;

    CAN_TxHeader.IdType              = FDCAN_STANDARD_ID;
    CAN_TxHeader.TxFrameType         = FDCAN_DATA_FRAME;
    CAN_TxHeader.DataLength          = FDCAN_DLC_BYTES_8;
    CAN_TxHeader.ErrorStateIndicator = FDCAN_ESI_ACTIVE;
    CAN_TxHeader.BitRateSwitch       = FDCAN_BRS_OFF;
    CAN_TxHeader.FDFormat            = FDCAN_CLASSIC_CAN;
    CAN_TxHeader.TxEventFifoControl  = FDCAN_NO_TX_EVENTS;
    CAN_TxHeader.MessageMarker       = 0;
    CAN_TxHeader.Identifier          = canID;

    switch(canID) {
        case CAN_ID_REMOTE_SENSOR:   // 0x100
            // BIT_FIELD에 데이터 채우기
            RemoteCanMsg.REMOTE_SENSOR.BIT_FIELD.BendingSensor_0
                = remoteData.bendingSensor[0];
            RemoteCanMsg.REMOTE_SENSOR.BIT_FIELD.BendingSensor_1
                = remoteData.bendingSensor[1];
            RemoteCanMsg.REMOTE_SENSOR.BIT_FIELD.GyroPitch
                = remoteData.gyro_pitch;
            RemoteCanMsg.REMOTE_SENSOR.BIT_FIELD.GyroRoll
                = remoteData.gyro_roll;

            // BYTE_FIELD로 송신
            for(i = 0; i < 8; i++) {
                CAN_TxData[i] =
                    RemoteCanMsg.REMOTE_SENSOR.BYTE_FIELD[i];
            }
            break;

        case CAN_ID_REMOTE_STATUS:   // 0x102
            RemoteCanMsg.REMOTE_STATUS.BIT_FIELD.RemoteCommStatus
                = sysStatus.canStatus;
            RemoteCanMsg.REMOTE_STATUS.BIT_FIELD.RemoteSysVolt
                = sysStatus.sysVolt;
            RemoteCanMsg.REMOTE_STATUS.BIT_FIELD.RemoteChecksum
                = CAN_CalcChecksum(
                    RemoteCanMsg.REMOTE_SENSOR.BYTE_FIELD, 8);

            for(i = 0; i < 8; i++) {
                CAN_TxData[i] =
                    RemoteCanMsg.REMOTE_STATUS.BYTE_FIELD[i];
            }
            break;

        default:
            return HAL_ERROR;
    }

    return HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
               &CAN_TxHeader, CAN_TxData);
}

        case CAN_ID_ROBOT_STATUS:    // 0x202
            RobotCanMsg.ROBOT_STATUS.BIT_FIELD.MotorStatus
                = sysStatus.motorStatus[0];
            RobotCanMsg.ROBOT_STATUS.BIT_FIELD.RobotCommStatus
                = sysStatus.canStatus;
            RobotCanMsg.ROBOT_STATUS.BIT_FIELD.RobotSysVolt
                = sysStatus.sysVolt;
            RobotCanMsg.ROBOT_STATUS.BIT_FIELD.LcdStatus
                = sysStatus.lcdStatus;
            RobotCanMsg.ROBOT_STATUS.BIT_FIELD.RobotChecksum
                = CAN_CalcChecksum(
                    RobotCanMsg.ROBOT_MOTOR_1.BYTE_FIELD, 8);

            for(i = 0; i < 8; i++) {
                CAN_TxData[i] =
                    RobotCanMsg.ROBOT_STATUS.BYTE_FIELD[i];
            }
            break;

        default:
            return HAL_ERROR;
    }

    return HAL_FDCAN_AddMessageToTxFifoQ(&hfdcan1,
               &CAN_TxHeader, CAN_TxData);
}

/****************************************************************
    Callback: HAL_FDCAN_RxFifo0Callback
    Description: CAN 수신 인터럽트 콜백
****************************************************************/
void HAL_FDCAN_RxFifo0Callback(FDCAN_HandleTypeDef *hfdcan,
                                uint32_t RxFifo0ITs) {
    if(hfdcan->Instance == FDCAN1) {
        FDCAN_RxHeaderTypeDef rxHeader;
        uint8_t rxData[8] = {0};

        HAL_FDCAN_GetRxMessage(hfdcan,
            FDCAN_RX_FIFO0, &rxHeader, rxData);

        lastCanRxTick = HAL_GetTick();
        canConnected  = 1;

        // ID 범위로 Fill 함수 분기
        if(rxHeader.Identifier >= 0x100 &&
           rxHeader.Identifier <= 0x1FF) {
            Fill_Remote_CAN_Message(
                rxHeader.Identifier, rxData);

        } else if(rxHeader.Identifier >= 0x200 &&
                  rxHeader.Identifier <= 0x2FF) {
            Fill_Robot_CAN_Message(
                rxHeader.Identifier, rxData);
        }
    }
}

전체 흐름 정리
수신 흐름:
인터럽트 콜백
    → Fill_Remote/Robot_CAN_Message()  RAW 구조체 채움
    → Update_RemoteData()              공용 구조체 변환
    → remoteData, motorData 바로 사용 가능

송신 흐름:
remoteData, motorData 업데이트
    → Tx_Remote/Robot_CAN_Message(canID)
    → BIT_FIELD 채우기
    → BYTE_FIELD로 전송
